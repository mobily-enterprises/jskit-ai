[shared contract cleanup]

Goal: remove the remaining mixed shared-contract schema styles now that `json-rest-schema` supports nested child schemas, array item schemas, and narrow opaque object bags.

Rules:
- shared `resources/` and `commands/` should use `json-rest-schema` contracts only
- raw JSON Schema stays in transport/infrastructure code only
- opaque bags are acceptable only for truly external or metadata-like payloads
- no role catalogs or other app-owned structured payloads as opaque object bags

[infra validation cleanup]

Goal: keep `json-rest-schema` as the authored validation system in infra too, while leaving pure transport/error envelope JSON Schema in place where it is genuinely transport-native.

Rules:
- authored query/body/params helpers should use `json-rest-schema`
- fake open-object validators should be removed instead of preserved
- transport-only JSON Schema is acceptable in envelope/error/merge utilities
