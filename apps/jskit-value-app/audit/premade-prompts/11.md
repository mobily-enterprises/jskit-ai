# Standard Prompt Template (State-Driven Auto/Audit/Fix):

Use this template for any entry in `audit/auditList.md`.

```txt
Read these files fully and follow them strictly:
- /home/merc/Development/current/jskit-ai/apps/jskit-value-app/audit/auditList.md
- /home/merc/Development/current/jskit-ai/apps/jskit-value-app/audit/instructions-auditing.md
- /home/merc/Development/current/jskit-ai/apps/jskit-value-app/audit/instructions-fixing.md

MODE:
- AUTO (recommended)
# or
- AUDIT_PASS
# or
- FIXING_PASS

TARGET_AUDIT_ENTRY
## 11) Alerts domain
Report file:
- `/home/merc/Development/current/jskit-ai/apps/jskit-value-app/audit/reports/11-alerts-domain.report.md`

Required scope:
- `/home/merc/Development/current/jskit-ai/apps/jskit-value-app/server/modules/alerts`
- `/home/merc/Development/current/jskit-ai/apps/jskit-value-app/src/views/alerts`
- `/home/merc/Development/current/jskit-ai/apps/jskit-value-app/server/runtime/actions/contributors/alerts.contributor.js`


Execution rules:
1) Resolve the exact entry and its report file path.
2) Ignore `audit/premade-prompts/**` unless explicitly asked to maintain prompt assets.

3) If MODE=AUTO:
   - Read report file first (if missing, treat as `WAITING_FOR_AUDIT`).
   - Determine state from `## Report state`.
   - If state is `WAITING_FOR_AUDIT`:
     - Run AUDIT_PASS behavior only.
     - If issues are found, set state to `BEING_FIXED`.
   - If state is `BEING_FIXED`:
     - Run FIXING_PASS behavior only.
     - If `Broken things` is empty, set state to `WAITING_FOR_AUDIT` and stop (no audit in same run).
   - Never switch domains.

4) If MODE=AUDIT_PASS:
   - Audit only that domain.
   - Read all required scope/docs from the entry.
   - Read related `tests/**` files for the same domain.
   - Create/update report file using instructions-auditing.md.
   - Do not remove history from `Fixed things` or `Won't fix things`.
   - Run targeted runtime validation commands when possible; if not possible, state why.
   - Apply cross-domain dedupe checks against existing `audit/reports/*.report.md`.
   - Do not make code edits.

5) If MODE=FIXING_PASS:
   - Read report file first.
   - Respect report state machine in instructions-fixing.md.
   - Choose one target issue and propose a fix plan.
   - Wait for explicit user approval token: `APPROVE NN-ISSUE-###` (example: `APPROVE 02-ISSUE-004`).
   - Do not edit code before explicit approval.
   - After approval, implement and validate fix.
   - Move issue to `Fixed things`/`Won't fix things` accordingly.
   - Create exactly one commit per fixed issue (`fix(audit): NN-ISSUE-### short-summary`).
   - Update linked issues in other reports when cross-domain duplicates exist.

6) Save report file.

7) Return a concise summary with:
   - report path
   - report state after execution
   - issue IDs changed
   - remaining broken count by severity
   - tests reviewed/run
   - runtime validation commands run (or not run + reason)
   - commit hash per fixed issue (if any)

If TARGET_AUDIT_ENTRY is ambiguous, ask one clarification question and stop.
Do not work on any other audit entry.
```
