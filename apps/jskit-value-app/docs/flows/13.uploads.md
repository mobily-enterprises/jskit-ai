# Uploads Flow Tutorial (Chat Attachments)

This tutorial follows attachment upload lifecycle.

Main endpoints:
- `POST /api/v1/chat/threads/:threadId/attachments/reserve`
- `POST /api/v1/chat/threads/:threadId/attachments/upload`
- `GET /api/v1/chat/attachments/:attachmentId/content`

## 1) Reserve step creates staged attachment

Controller:

```js
// chat-fastify-adapter controller
async function reserveThreadAttachment(request, reply) {
  const result = await executeAction(actionExecutor, {
    actionId: CHAT_ACTION_IDS.ATTACHMENT_RESERVE,
    request,
    input: {
      threadId: request.params?.threadId,
      payload: request.body || {}
    }
  });

  reply.code(200).send(result);
}
```

Service core:

```js
// chat-core service
const existing = await chatAttachmentsRepository.findByClientAttachmentId(...);
if (existing) {
  if (!isAttachmentReserveReplayCompatible(existing, normalizedPayload)) {
    throw createAttachmentConflictError("clientAttachmentId conflicts with existing attachment metadata.");
  }
  return { attachment: mapAttachmentForResponse(existing) };
}

createdAttachment = await chatAttachmentsRepository.insertReserved({ ... });
```

## 2) Upload step validates multipart + size

```js
// chat-fastify-adapter controller
const uploadPayload = await readOneMultipartAttachment(request, maxUploadBytes, AppErrorClass);
const result = await executeAction(actionExecutor, {
  actionId: CHAT_ACTION_IDS.ATTACHMENT_UPLOAD,
  request,
  input: {
    threadId: request.params?.threadId,
    attachmentId: uploadPayload.attachmentId,
    payload: uploadPayload
  }
});
```

`readOneMultipartAttachment(...)` enforces:
- multipart required
- single file
- max file size

## 4) Storage implementation (filesystem driver)

```js
// packages/chat/chat-storage-core/src/attachmentStorage.service.js
const storageKey = buildAttachmentStorageKey({ threadId, attachmentId, fileName });
await storage.setItemRaw(storageKey, buffer);
return { storageKey };
```

Storage key format:
- `chat/threads/<threadId>/attachments/<attachmentId>/<uuid>-<safeFileName>`

## 5) Content read flow

```js
// chat-core service
const contentBuffer = await chatAttachmentStorageService.readAttachment(storageKey);
...
return {
  attachment: mapAttachmentForResponse(attachment),
  contentBuffer,
  contentType,
  contentDisposition
};
```

Controller sets security headers:
- `Cache-Control: private, no-store`
- `Vary: Authorization, Cookie`
- `X-Content-Type-Options: nosniff`

## 6) Realtime updates

Attachment reserve/upload/delete publish `chat.attachment.updated` style events so clients can update staged state quickly.
