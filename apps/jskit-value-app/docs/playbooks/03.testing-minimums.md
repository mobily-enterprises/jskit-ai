# Testing Minimums Playbook

Use this matrix when deciding what tests must be added.

## Minimum By Change Type

1. Route/schema changes.
- Route contract test (status + payload shape).
- Validation failure test (`400` + `fieldErrors`).
- Policy test (`401`/`403` where applicable).

2. Action definition changes.
- Contributor/action registry test coverage for definition validity.
- Permission behavior test for allowed/denied actors.
- Idempotency behavior test if policy is `optional`/`required`/`domain_native`.

3. Service logic changes.
- Unit tests for normalization + invariants.
- Error-case tests for expected `AppError` payloads.

4. Repository/data changes.
- Integration tests against DB behavior.
- Transaction and edge case tests (duplicates, missing rows, race-safe flows).

5. Realtime mutation changes.
- Verify publish-on-success behavior.
- Verify no publish on failure path.

6. Permission model changes.
- Route/action permission parity tests.
- Surface/channel gating tests for forbidden operations.

## Always Run (Project Rails)

- `npm --prefix ../.. run lint:architecture:client`
- `npm --prefix ../.. run test:architecture:client`
- `npm --prefix ../.. run test:architecture:shared-ui`
- `npm run lint`
- `npm run test`
- `npm run test:client`

Plus targeted suites for affected domains (billing/realtime/worker/auth as needed).
