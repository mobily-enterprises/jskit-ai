# Q5: Hidden global state — assistant-core service mutates module-level overrides

## Finding
- Location: `packages/ai-agent/assistant-core/src/shared/service.js:29-116, 821-847`
- Behavior: Module-level overrides (`AppError`, `safePathnameFromRequest`, `publishWorkspaceEventSafely`, etc.) mutated in `createService`.

## Why risky
Multiple service instances race and overwrite each other; one instance can affect another’s behavior.

## Suggested fix
Store overrides in closure variables inside `createService` and pass them into helpers via local `deps`.
