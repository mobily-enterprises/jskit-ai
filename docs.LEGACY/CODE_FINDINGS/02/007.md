# Q2: Fake architecture — realtime event bus global singleton

## Finding
A global event bus is introduced for just two subscribers.

- Definition: `apps/jskit-value-app/src/platform/realtime/realtimeEventBus.js:1-46`.
- Publisher: `apps/jskit-value-app/src/platform/realtime/realtimeEventHandlers.js:348`.
- Subscribers: `apps/jskit-value-app/src/modules/chat/runtime.js:7-12` and `apps/jskit-value-app/src/app/state/alertsStore.js:3`.
- Usage count: 1 publish, 2 subscribe.

## Why it’s fake
A global mutable event bus adds indirection and hidden coupling for a tiny fan‑out. This looks like an event‑bus pattern for its own sake.

## Suggested refactor
Pass callbacks directly from realtime runtime into chat and alerts, or keep a small in‑module observer list inside `realtimeEventHandlers` rather than a shared global bus.
