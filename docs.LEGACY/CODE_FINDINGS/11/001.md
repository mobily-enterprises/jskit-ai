# Finding 001
- Severity: High
- Pattern type: Hidden transitive coupling; Module dependency drift
- File + line(s): `apps/jskit-value-app/server/framework/moduleRegistry.js:389-396`, `apps/jskit-value-app/server/runtime/actions/contributors/deg2radHistory.contributor.js:113-126`
- Why it can drift/fail: The `deg2rad` module contributes `actionContributorModules: ["history"]` but does not declare a module dependency on `history`. The action contributor requires `deg2radHistoryService`, which is owned by the `history` module. This creates a hidden, non-declared runtime coupling that only surfaces when a minimal profile excludes `history`.
- Concrete impact: A calculator-focused or minimal server profile can omit the `history` module and still pass registry validation, then fail during action contributor initialization because `deg2radHistoryService` is missing.
- Exact remediation: Add `dependsOnModules: Object.freeze([moduleDependency("history")])` to the `deg2rad` module definition or remove the `history` action contributor from `deg2rad` if the dependency is not required. Add a boot test for calculator-only profiles to catch this class of hidden dependency.
