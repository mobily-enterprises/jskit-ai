# Q4: Convenient lies — bootstrap failure looks like clean sign-out

## Finding
- Location: `apps/jskit-value-app/src/app/bootstrap/runtime.js:166-170`
- Behavior: Catch-all boot failure resets auth/workspace state without surfacing underlying error.

## Why risky
Users see what looks like a clean sign-out; root cause is lost, debugging is harder.

## Better error strategy
Log the error (console + observability) and present a specific “Bootstrap failed, please retry” message instead of silently clearing.
