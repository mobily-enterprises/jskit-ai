# Q3: Contract drift — chat `sendMessage` allows empty content

## Finding
- Contract: `packages/chat/chat-fastify-routes/src/shared/schema.js:86-98` allows both `text` and `attachmentIds` to be optional with no cross-field constraint.
- Runtime: `packages/chat/chat-core/src/shared/service.js:503-517` rejects when both are missing.

## Drift
Schema allows empty messages; runtime requires either text or attachments.

## Risk
Requests that validate against the schema still 400 at runtime, creating confusing client errors.

## Suggested fix
Add a schema constraint (`anyOf` requiring `text` or `attachmentIds`, or `dependentRequired`) or relax runtime to allow empty messages if that is intended.
