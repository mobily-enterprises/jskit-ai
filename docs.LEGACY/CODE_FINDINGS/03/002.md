# Q3: Contract drift — `simulateServerError` success payload never returned

## Finding
- Contract: `packages/workspace/console-errors-fastify-routes/src/shared/schema.js:153-160` expects `{ ok, simulationId, kind }`.
- Runtime: `packages/observability/observability-core/src/shared/consoleErrors.service.js:189-215` always throws/rejects.

## Drift
The schema promises a success payload, but the service always fails.

## Risk
Clients may implement success handling paths that can never be hit; contract validation is misleading.

## Suggested fix
Update schema to document error-only behavior, or add a success path that returns `{ ok: true, simulationId, kind }` and reserve throws for explicit failure cases.
