# Q10: Invariant not asserted — client runtime config must be validated

## Finding
- Locations:
  - `apps/jskit-value-app/src/modules/assistant/runtime.js:2-15`
  - `apps/jskit-value-app/src/modules/chat/runtime.js:3-21`
  - Config validation occurs in `apps/jskit-value-app/config/index.js:38-63`

## Invariant assumed
Client runtime config is always valid and properly overridden.

## Why risky
Direct imports bypass validators and override logic; malformed config slips through.

## Suggested fix
Export validated config from `config/index.js` (or a `getClientConfig()` helper) and use that in runtimes. Add a test or lint rule preventing direct imports of raw config files in client runtime code.
