# Docs Contribution Guide

This repository treats manual chapters and `docs/examples/*` as one system.

## Rules

- Every manual chapter must reference at least one runnable example package in `docs/examples`.
- Code snippets that should stay in sync with source files must use `DOCS:SYNC` blocks.
- The canonical mapping is `docs/examples/manifest.json`.

## Required workflow

1. Create or update example code under `docs/examples/*`.
2. Reference the example package path in the matching manual chapter.
3. If a code block should be synced from source, wrap it in a `DOCS:SYNC` block.
4. Run:
   - `npm run docs:sync`
   - `npm run docs:verify`

## `DOCS:SYNC` block format

````md
<!-- DOCS:SYNC source="docs/examples/03.real-app/src/server/snippets/ch03DocSnippets.js" snippet="formrequest_route_upgrade" lang="js" -->
```js
// this body is generated by npm run docs:sync
```
<!-- /DOCS:SYNC -->
````

Snippet source files must declare markers:

```js
// docs:start:formrequest_route_upgrade
// ... snippet body ...
// docs:end:formrequest_route_upgrade
```

## `DOCS:EXAMPLE` block format

Use this when the full code block should come directly from a runnable docs example file.

Controller example (requested format):

````md
<!-- DOCS:EXAMPLE package="03.real-app" controller="ContactControllerStage2" lang="js" -->
```js
// this body is generated by npm run docs:sync
```
<!-- /DOCS:EXAMPLE -->
````

Supported selectors:

- `controller="Name"`
- `provider="Name"`
- `service="Name"`
- `action="Name"`
- `repository="Name"`
- `schema="Name"`
- `file="relative/path/from/example/package"`

`package` can be either:

- short form, e.g. `03.real-app` (resolved under `docs/examples/`)
- full form, e.g. `docs/examples/03.real-app`

## Verification behavior

`npm run docs:verify` checks:

- manifest structure
- chapter file existence
- example package existence and required file shape
- chapter references to mapped example package paths
- mapped source file existence
- snippet sync status (`docs:sync --check`)

If verification fails, update the chapter/example/manifest mapping or run `npm run docs:sync`.
