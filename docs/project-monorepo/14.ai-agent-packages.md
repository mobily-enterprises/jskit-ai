# AI Agent Package Architecture (`@jskit-ai/assistant-*`)

Status: Implemented  
Date: 2026-02-24

## 1. Goal

Share assistant orchestration, transcripts handling, provider integration, and adapter/runtime contracts while keeping tool policy, role vocabulary, and UI behavior app-owned.

## 2. Package Topology

1. `@jskit-ai/assistant-contracts`
   1. Assistant query keys.
   2. Transcript query keys.
   3. Stream event constants and normalizers.
2. `@jskit-ai/assistant-core`
   1. Assistant orchestration service.
   2. Tool registry mechanics.
3. `@jskit-ai/assistant-transcripts-core`
   1. Transcript mode resolution.
   2. Transcript redaction helpers.
   3. Transcript service behavior.
4. `@jskit-ai/assistant-provider-openai`
   1. OpenAI provider client adapter.
5. `@jskit-ai/assistant-client-runtime`
   1. Headless assistant composable runtime (`useAssistantRuntime`).
   2. Assistant HTTP API builder (`createApi` / `createAssistantApi`).
6. `@jskit-ai/assistant-fastify-adapter`
   1. Fastify AI route schema/controller/route builders.
   2. NDJSON stream writer helpers.

## 3. Boundaries

1. Contracts package has no app imports and no UI/framework transport logic.
2. Core packages (`assistant-core`, `assistant-transcripts-core`) own business rules, not HTTP concerns.
3. Fastify adapter owns HTTP/stream transport concerns only.
4. Client runtime package owns headless state logic, not Vue SFC rendering/styling.
5. Packages must not import from `apps/*`.

## 4. App-Owned Customization

1. `apps/jskit-value-app/server/modules/ai/tools/workspaceRename.tool.js` stays app-local.
2. App wrapper `apps/jskit-value-app/server/modules/ai/service.js` injects:
   1. App `AppError`.
   2. App permission checks.
   3. App route/surface helpers.
   4. App realtime topics/event constants.
   5. App prompt resolvers.
   6. App tool allowlist defaults.
3. App transcript wrapper injects console permission vocabulary and role resolution.
4. `apps/jskit-value-app/src/views/assistant/AssistantView.vue` remains app-local.

## 5. Compatibility Wrappers

Wrappers preserve app import stability while delegating implementation:

1. `apps/jskit-value-app/src/features/assistant/queryKeys.js`
2. `apps/jskit-value-app/src/features/aiTranscripts/queryKeys.js`
3. `apps/jskit-value-app/src/services/api/aiApi.js`
4. `apps/jskit-value-app/src/views/assistant/useAssistantView.js`
5. `apps/jskit-value-app/server/modules/ai/schema.js`
6. `apps/jskit-value-app/server/modules/ai/routes.js`
7. `apps/jskit-value-app/server/modules/ai/controller.js`
8. `apps/jskit-value-app/server/modules/ai/stream/ndjson.js`
9. `apps/jskit-value-app/server/modules/ai/service.js`

## 6. Extension Points

1. Add new providers by creating `assistant-provider-*` packages consumed by `assistant-core` wiring wrappers.
2. Add app/product tools in app-local tool modules and inject into assistant-core wrappers.
3. Add transport adapters (non-Fastify) on top of `assistant-core` and `assistant-contracts`.
4. Keep surface-specific policy and permission vocabularies app-owned.

## 7. Migration Notes

1. Assistant service orchestration moved to `@jskit-ai/assistant-core`.
2. Transcript mode/redaction/service moved to `@jskit-ai/assistant-transcripts-core`.
3. OpenAI provider client moved to `@jskit-ai/assistant-provider-openai`.
4. AI routes/controller/schema/ndjson helpers moved to `@jskit-ai/assistant-fastify-adapter`.
5. Assistant client API/composable runtime moved to `@jskit-ai/assistant-client-runtime`.
6. App wrappers retain injection points for app policy and tool ownership.
