# Chat Package Architecture (`@jskit-ai/chat-*`)

Status: Implemented  
Date: 2026-02-24

## 1. Goal

Share chat contracts/runtime/server core across apps while keeping product policy and UI presentation in each app.

## 2. Package Topology

1. `@jskit-ai/chat-contracts`
   1. Query keys and chat error mapping primitives.
2. `@jskit-ai/chat-core`
   1. Chat service business rules.
   2. Generic realtime publish service behavior.
3. `@jskit-ai/chat-client-runtime`
   1. Headless chat composable runtime (`useChatRuntime`).
   2. Chat HTTP API builder (`createApi` / `createChatApi`).
4. `@jskit-ai/chat-fastify-adapter`
   1. Fastify route schema/controller/route builders for chat endpoints.

## 3. Boundaries

1. `chat-contracts` has no Fastify/Vue imports.
2. `chat-core` has no HTTP/controller concerns.
3. `chat-fastify-adapter` handles HTTP transport concerns only.
4. `chat-client-runtime` owns headless state/flows, not SFC rendering/styling.
5. Packages must not import from `apps/*`.

## 4. App-Owned Customization

1. `apps/jskit-value-app/src/views/chat/ChatView.vue` remains app-local.
2. App wrapper `apps/jskit-value-app/src/views/chat/useChatView.js` injects:
   1. App API object.
   2. Auth guard composable.
   3. Query error helper.
   4. Workspace store.
   5. Realtime event bus + event type catalog.
3. App chat module wrappers inject `AppError` and app realtime constants.

## 5. Compatibility Wrappers

Current wrappers keep legacy import paths stable while delegating to packages:

1. `apps/jskit-value-app/src/features/chat/queryKeys.js`
2. `apps/jskit-value-app/src/features/chat/errors.js`
3. `apps/jskit-value-app/src/services/api/chatApi.js`
4. `apps/jskit-value-app/src/views/chat/useChatView.js`
5. `apps/jskit-value-app/server/modules/chat/schema.js`
6. `apps/jskit-value-app/server/modules/chat/routes.js`
7. `apps/jskit-value-app/server/modules/chat/controller.js`
8. `apps/jskit-value-app/server/modules/chat/service.js`
9. `apps/jskit-value-app/server/domain/chat/services/realtime.service.js`

## 6. Extension Points

1. Add chat transport variants by creating new adapters (for example non-Fastify) on top of `chat-core` and `chat-contracts`.
2. Add app-specific chat policy in wrappers only.
3. Keep UI variants app-local and consume `useChatRuntime` contracts.

## 7. Migration Notes

1. Old app-local chat API/composable logic moved into `@jskit-ai/chat-client-runtime`.
2. Old chat service/realtime service logic moved into `@jskit-ai/chat-core`.
3. Old chat route/controller/schema logic moved into `@jskit-ai/chat-fastify-adapter`.
4. Query keys and error mapping now come from `@jskit-ai/chat-contracts`.
