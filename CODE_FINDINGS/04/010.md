# Q4: Convenient lies — social feed errors labeled “blocked”

## Finding
- Location: `packages/social/social-core/src/shared/service.js:574-612`
- Behavior: URL parsing/DNS failures are reported as “Blocked remote host.”

## Why risky
Misleading error messages hide the real issue (invalid URL, DNS failure), leading to wrong remediation.

## Better error strategy
Return distinct error codes (`invalid_url`, `dns_failed`, `blocked`) while logging the underlying exception.
