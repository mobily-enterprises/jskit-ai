# Q4: Convenient lies — worker interval errors swallowed

## Finding
- Location: `packages/billing/billing-worker-core/src/shared/workerRuntime.service.js:229-234`
- Behavior: Initial interval run errors are swallowed (`run().catch(() => {})`), timers continue silently.

## Why risky
Jobs can fail immediately with no trace while intervals keep running, leading to false “healthy” behavior.

## Better error strategy
Log/metric per interval name and store last-error marker; optionally rethrow to fail fast in non-prod.
