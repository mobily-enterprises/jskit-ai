# Q5: Hidden global state — dotenv loads mutate process.env

## Finding
- Location: `packages/runtime/runtime-env-core/src/shared/platformRuntimeEnv.js:23-63`
- Behavior: `loadDotenvFiles` mutates `process.env` and defaults to `loadDotenv=true`, based on `process.cwd`.

## Why risky
Order-dependent environment in tests and multi-app processes; globals mutate implicitly.

## Suggested fix
Only call dotenv loading at entrypoints; pass an env object into `createPlatformRuntimeEnv`.
