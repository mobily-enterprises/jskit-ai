# Q5: Hidden global state — alerts store singleton listeners

## Finding
- Location: `apps/jskit-value-app/src/app/state/alertsStore.js:14-20, 198-308`
- Behavior: Module-level `sync*` listeners, `realtimeAlertsUnsubscribe`, and `TAB_ID` create global singletons.

## Why risky
Multiple store instances share listeners; hot reload leaves stale handlers; tests can double-subscribe.

## Suggested fix
Move listeners into store instance state, or create `createAlertsSyncController()` with attach/detach injected into the store. Ensure cleanup on disposal.
