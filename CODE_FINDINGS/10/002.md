# Q10: Invariant not asserted — navigation fragment IDs unique

## Finding
- Locations:
  - `packages/web/web-runtime-core/src/shared/clientComposition.js:395-425` collects fragments without dedupe.
  - `apps/jskit-value-app/src/framework/composeNavigation.js:27-39` only checks collisions between module and filesystem entries.

## Invariant assumed
Navigation fragment IDs are unique across all modules.

## Why risky
Duplicate IDs can cause ambiguous routing/ordering and subtle UI bugs.

## Suggested fix
Add duplicate ID (and path) checks in `composeNavigationFragmentsFromModules` and unit tests for collisions.
