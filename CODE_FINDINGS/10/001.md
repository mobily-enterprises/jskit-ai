# Q10: Invariant not asserted — route mount filtering uses wrong key

## Finding
- Locations:
  - `packages/web/web-runtime-core/src/shared/clientComposition.js:54-62` filters by `entry.id`.
  - Route mounts are defined with `moduleId` only at `apps/jskit-value-app/src/framework/routeMountRegistry.js:1-43`.
  - `apps/jskit-value-app/src/framework/composeRouteMounts.js:15-21` passes `enabledModuleIds`.

## Invariant assumed
Route mount entries have `id` that matches `enabledModuleIds`.

## Why risky
When `enabledModuleIds` is provided, all mounts are dropped because they only have `moduleId`.

## Suggested fix
Map `moduleId` → `id` for mount entries or update `resolveActiveClientModules` to accept `entry.moduleId`. Add a test that `enabledModuleIds` still yields mounts.
