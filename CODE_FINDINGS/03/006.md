# Q3: Contract drift — SMS provider enum too strict

## Finding
- Contract: `packages/communications/communications-contracts/src/shared/schema.js:8-9` and `:22-31` restrict provider to `none | plivo`.
- Runtime: `packages/communications/communications-core/src/shared/orchestrator.js:41-78` returns provider from dispatch, which can be custom/other values.

## Drift
Schema allows only a small enum, but runtime may return other provider identifiers.

## Risk
Response validation fails when custom SMS providers are used.

## Suggested fix
Loosen provider schema to `Type.String({ minLength: 1 })` or normalize provider names to the enum.
