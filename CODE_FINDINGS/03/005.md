# Q3: Contract drift — `sendEmail` response includes extra field

## Finding
- Contract: `packages/communications/communications-contracts/src/shared/schema.js:47-56` has `additionalProperties: false` (no `request` field).
- Runtime: `packages/communications/email-core/src/shared/service.js:14-26` returns `request` with `to`, `subject`, `metadata`.

## Drift
Runtime includes a `request` field that the schema forbids.

## Risk
Response validation failures and undocumented fields exposed to clients.

## Suggested fix
Remove `request` from the response (keep internal/log-only), or add it to the schema and document it explicitly.
