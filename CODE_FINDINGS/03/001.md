# Q3: Contract drift — console errors `recordBrowserError` response shape mismatch

## Finding
- Contract: `packages/workspace/console-errors-fastify-routes/src/shared/schema.js:133-139` expects `{ ok: boolean }`.
- Runtime: `packages/observability/observability-core/src/shared/consoleErrors.service.js:153-162` returns the repository insert result; `packages/workspace/workspace-console-service-core/src/shared/repositories/errorLogs.repository.js:96-118` maps a full entry object.

## Drift
The contract claims a simple `{ ok }` response, but runtime returns a full error log entry (`id`, `createdAt`, `message`, etc.).

## Risk
Response validation failures and undocumented API behavior for clients expecting `{ ok: boolean }`.

## Suggested fix
Either change the response schema to the entry object or update the handler to return `{ ok: true }` and expose entry details via a separate endpoint.
