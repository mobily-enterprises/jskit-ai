# Q7: Performance — unbounded fanout concurrency in realtime server

## Finding
- Location: `packages/realtime/realtime-server-socketio/src/shared/registerRealtimeServerSocketio.js:1370, 1422, 1479, 1535, 1634`
- Behavior: `Promise.all` across `targetUserIds`, then across all sockets per event.

## Impact
Large fanouts can spike CPU/memory and overload Socket.IO. Single failures can block entire batches.

## Minimal refactor
Introduce a concurrency cap (e.g., `p-limit`) for socket lookup and per-socket emits. Batch `targetUserIds`/`sockets` (100–500) and await per batch.
