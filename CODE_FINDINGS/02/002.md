# Q2: Fake architecture — client compose wrappers are thin pass‑throughs

## Finding
Client “compose” wrappers add a layer with a static registry and 1–3 call sites.

- Definition: `apps/jskit-value-app/src/framework/composeApi.js:8`, `composeGuards.js:4`, `composeNavigation.js:9`, `composeRouter.js:35`, `composeRealtimeClient.js:7`.
- Callers:
  - `composeClientApi` used once at `apps/jskit-value-app/src/platform/http/api/index.js:4`.
  - `composeGuardPolicies` used at `apps/jskit-value-app/src/app/router/app.js:9`, `apps/jskit-value-app/src/app/router/admin.js:9`, and fallback in `apps/jskit-value-app/src/app/router/guards.js:47`.
  - `composeSurfaceRouteFragments` used at `apps/jskit-value-app/src/app/router/app.js:8`, `apps/jskit-value-app/src/app/router/admin.js:8`, `apps/jskit-value-app/src/app/router/console.js:23`.
  - `composeNavigationFragments` used at `apps/jskit-value-app/src/app/shells/app/useAppShell.js:116` and `apps/jskit-value-app/src/app/shells/admin/useAdminShell.js:140`.
  - `composeRealtimeTopicContributions` used at `apps/jskit-value-app/src/platform/realtime/realtimeRuntime.js:85`.
  - `composeRealtimeInvalidationDefinitions` used at `apps/jskit-value-app/src/platform/realtime/realtimeEventHandlers.js:238`.
- Usage count: each wrapper has 8–11 references in `apps/jskit-value-app/src`, but only 1–3 runtime call sites.

## Why it’s fake
This is extra indirection for a static module list, without any feature gating, plugin loading, or runtime swapping. The abstraction doesn’t buy flexibility here.

## Suggested refactor
Inline the small outputs as constants (routes, nav fragments, guard policies, topics) or call the core `compose*FromModules` directly in the 1–3 call sites. Remove the wrapper layer.
