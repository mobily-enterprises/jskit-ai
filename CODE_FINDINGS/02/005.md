# Q2: Fake architecture — server app extensions loaded but empty

## Finding
Server app extension descriptors are loaded but contribute nothing.

- Definition: `apps/jskit-value-app/server/app/loadExtensions.server.js:7` and extensions `apps/jskit-value-app/server/app/extensions.d/10-projects.server.js:1-11`, `apps/jskit-value-app/server/app/extensions.d/20-realtime.server.js:1-11`.
- Caller: `apps/jskit-value-app/server.js:97` loads `APP_SERVER_EXTENSIONS` and passes into runtime.

## Why it’s fake
Startup cost + validation from `@jskit-ai/module-framework-core`, but every server extension contributes empty arrays. This is a plugin system without plugins.

## Suggested refactor
Remove `extensions.d` loading (or delete these empty descriptors) until real extensions exist. Keep `settings.extensions.d` and `workers.extensions.d` only if those are actually used.
