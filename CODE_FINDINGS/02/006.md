# Q2: Fake architecture — provider registry service wrappers

## Finding
Billing provider registry “services” are thin wrappers over `createProviderRegistry` with no added value.

- Definition: `apps/jskit-value-app/server/modules/billing/lib/providers/shared/providerRegistry.service.js:7-34` and `apps/jskit-value-app/server/modules/billing/lib/providers/shared/webhookTranslationRegistry.service.js:7-37`.
- Caller: `apps/jskit-value-app/server/modules/billing/lib/providers/index.js:122-137` (plus tests).
- Usage count: 1 production caller each.

## Why it’s fake
Extra service objects only forward to the underlying registry and add minor error message text. No real abstraction benefit.

## Suggested refactor
Use `createProviderRegistry` directly inside `apps/jskit-value-app/server/modules/billing/lib/providers/index.js` and delete the two wrapper service modules.
