# Q8: Async correctness — potential listener leak in profile avatar uploader

## Finding
- Location: `apps/jskit-value-app/src/views/settings/profile/lib/useSettingsProfileLogic.js:126`
- Behavior: Uppy listeners are registered but the instance is never closed on unmount.

## Risk
Listener leak and duplicated handlers over time, especially with HMR or repeated mounts.

## Suggested fix
Add `onBeforeUnmount(() => { uppy.close({ reason: "unmount" }); avatarUppy.value = null; })` or remove listeners via `uppy.off`.
