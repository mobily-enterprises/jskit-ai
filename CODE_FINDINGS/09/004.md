# Q9: Security — shell command execution with `shell: true`

## Finding
- Location: `packages/tooling/app-scripts/bin/jskit-app-scripts.js:108-114` builds a shell command string and spawns with `shell: true`, used for string tasks and `shell: true` tasks at `packages/tooling/app-scripts/bin/jskit-app-scripts.js:264-273`.

## Threat
If task config is untrusted, this is command injection by design.

## Fix
Default to `spawn(command, args, { shell: false })`, require explicit `allowShell: true` for shell execution, and warn or block when command contains shell metacharacters.
