# Finding 006
- Severity: Low
- Pattern type: God file; Wiring sprawl
- File + line(s): `apps/jskit-value-app/server/runtime/services.js:1-120`
- Why it can drift/fail: `services.js` centralizes many unrelated service constructors, exports, and wiring logic in one file. As the module ecosystem grows, this file becomes a high-conflict merge hotspot and a single choke point for service lifecycle changes.
- Concrete impact: Increases the chance of wiring regressions and makes incremental changes harder to review, because multiple domains are interleaved in one place.
- Exact remediation: Split wiring by domain (e.g., `runtime/services/billing.js`, `runtime/services/workspace.js`) and assemble them in a small aggregator. Keep the export ID list and creation policy in one place but delegate implementation to per-domain modules.
