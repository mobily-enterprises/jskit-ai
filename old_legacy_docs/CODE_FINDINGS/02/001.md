# Q2: Fake architecture — client extension registry is a no-op

## Finding
Client app extension loader and module registry merge add a plugin system with no real contributions.

- Definition: `apps/jskit-value-app/src/app/loadExtensions.client.js:15` (`loadClientAppExtensions`) and `apps/jskit-value-app/src/framework/moduleRegistry.js:5` (`resolveClientModuleRegistry`).
- Callers: `apps/jskit-value-app/src/framework/moduleRegistry.js:6-9` (merge), used only by compose wrappers (see other findings).
- Usage count: `getClientAppExtensions` has 1 runtime caller; client extension modules are empty objects in `apps/jskit-value-app/src/app/extensions.d/10-projects.client.js:1-8` and `apps/jskit-value-app/src/app/extensions.d/20-realtime.client.js:1-8`.

## Why it’s fake
Dynamic extension discovery + merge + caching, but every extension contributes empty arrays. This is composability without any actual extension use.

## Suggested refactor
Remove client extension loading/merging until real contributions exist. Export `CLIENT_MODULE_REGISTRY` directly from `apps/jskit-value-app/src/framework/moduleRegistry.base.js:25` and call `compose*FromModules` with that constant.
