# Q10: Invariant not asserted — navigation fragments must resolve to a path

## Finding
- Locations:
  - `packages/web/web-runtime-core/src/shared/clientComposition.js:405-420` allows fragments without `mountKey` and without `path`.
  - `apps/jskit-value-app/src/app/shells/app/useAppShell.js:118-125` falls back to `fragment.path || "/"`.

## Invariant assumed
Every navigation fragment resolves to a non-empty path.

## Why risky
Fragments missing both `mountKey` and `path` collapse to `/`, causing incorrect routing and collisions.

## Suggested fix
Validate that each fragment resolves to a non-empty path; disallow `/` unless explicitly intended. Add a test for a fragment missing both `mountKey` and `path`.
