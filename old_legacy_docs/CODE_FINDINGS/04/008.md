# Q4: Convenient lies — JSON parse errors masked as empty payload

## Finding
- Location: `packages/web/http-client-runtime/src/shared/client.js:42-48`
- Behavior: JSON parse errors are converted to `{}`; broken responses look like successful empty payloads.

## Why risky
Downstream logic treats parse failures as valid but empty data; real error cause is lost.

## Better error strategy
Capture parse error and return `{ data:{}, parseError }` or throw a typed `invalid_json` error.
