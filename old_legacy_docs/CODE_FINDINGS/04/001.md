# Q4: Convenient lies — alerts refresh errors swallowed

## Finding
- Location: `apps/jskit-value-app/src/app/state/alertsStore.js:459-472`
- Behavior: Initial refresh and polling swallow errors (`catch {}` and `.catch(() => {})`). UI proceeds as if alerts loaded.

## Why risky
Users see no indication that alerts failed to load; repeated failures are invisible and can cause stale UI without diagnostics.

## Better error strategy
Record a non-blocking warning in store state and log to observability (include error + context). Add backoff/last-error timestamp to avoid silent loops.
