# Q5: Hidden global state — client identity cached at module scope

## Finding
- Location: `apps/jskit-value-app/src/platform/realtime/clientIdentity.js:1-45`
- Behavior: Module-level `inMemoryClientId` fallback and sessionStorage-backed ID.

## Why risky
Shared ID across multiple app instances in the same JS context; tests/HMR reuse stale ID.

## Suggested fix
Inject a `clientIdProvider` and storage interface; create per-bootstrap instance.
