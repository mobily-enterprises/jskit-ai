# Chat Message Flow Tutorial

Example path:
- `POST /api/chat/threads/:threadId/messages`

## 1) Controller maps transport payload to action

```js
// packages/chat/chat-fastify-adapter/src/controller.js
async function sendThreadMessage(request, reply) {
  const result = await executeAction(actionExecutor, {
    actionId: CHAT_ACTION_IDS.THREAD_MESSAGE_SEND,
    request,
    input: {
      threadId: request.params?.threadId,
      payload: request.body || {}
    }
  });

  reply.code(200).send(result);
}
```

## 2) Action definition enforces channel/surface and domain-native idempotency intent

```js
// packages/chat/chat-core/src/actions/chat.contributor.js
{
  id: "chat.thread.message.send",
  channels: ["api", "assistant_tool", "internal"],
  surfaces: ["app", "admin", "console"],
  permission: requireAuthenticated,
  idempotency: "domain_native",
  async execute(input, context) {
    ...
    return chatService.sendThreadMessage({
      user: resolveUser(context, payload),
      threadId: resolveThreadId(payload),
      surfaceId: resolveSurfaceId(context, payload),
      payload: body,
      requestMeta: buildRequestMeta(context, payload)
    });
  }
}
```

Realtime topic policy alignment:
- `chat` and `typing` subscriptions are allowed on `app` and `admin` surfaces.

## 3) Service normalizes and fingerprints payload

```js
// packages/chat/chat-core/src/service.js
const normalizedPayload = normalizeSendPayload(payload, options);
```

## 4) Transaction branch: replay vs conflict vs create

```js
const existing = await chatMessagesRepository.findByClientMessageId(...);
if (existing) {
  const matches = await doesMessageMatchIdempotencyFingerprint(existing, incomingFingerprint, scopedOptions);
  if (!matches) {
    throw createIdempotencyConflictError();
  }

  return {
    idempotencyStatus: "replayed",
    messageId: existing.id,
    thread: replayThread
  };
}
```

Then it checks tombstones for deleted-message retry protection.

## 5) New-message branch writes message + attachments + read cursor

```js
createdMessage = await chatMessagesRepository.insert({
```

Then updates:

- participant read cursor

## 6) Realtime fanout

```js
realtimePublisher.publishMessageEvent({
  thread: responseThread,
  message: responsePayload.message,
  idempotencyStatus: responsePayload.idempotencyStatus,
  actorUserId: userId,
  targetUserIds: participantUserIds,
  commandId: normalizedRequestMeta.commandId,
  sourceClientId: normalizedRequestMeta.sourceClientId,
  workspaceSlug: normalizedRequestMeta.workspaceSlug
});
```

Delivery rules to remember for workspace chat events:
- Envelope carries `workspaceId` + `workspaceSlug`.
- Socket must have an active workspace-topic subscription for `chat`.
- Runtime re-checks authorization per socket before emit and evicts stale subscriptions on deny.

## Data shape to remember

Input payload usually includes:
- `clientMessageId` (required)
- `text`
- optional `attachmentIds`, `replyToMessageId`, `metadata`

Output includes:
- `message`
- `thread`
- `idempotencyStatus` (`created` or `replayed`)
