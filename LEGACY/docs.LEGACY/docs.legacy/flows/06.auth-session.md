# Auth Session Flow Tutorial

This walkthrough traces auth/session endpoints through actions.

Example endpoint:
- `POST /api/login`

## 1) Route -> controller mapping

```js
// packages/auth/auth-web/src/routes.js
{
  path: "/api/login",
  method: "POST",
  auth: "public",
  schema: { ... },
  handler: controllers.auth.login
}
```

## 2) Controller delegates to canonical action

```js
// packages/auth/auth-web/src/controller.js
async function login(request, reply) {
  const payload = request.body || {};
  const result = await executeAction(actionExecutor, {
    actionId: AUTH_ACTION_IDS.LOGIN_PASSWORD,
    request,
    input: payload
  });

  authService.writeSessionCookies(reply, result.session);

  reply.code(200).send({
    ok: true,
    username: result.profile.displayName
  });
}
```

Controller responsibility:
- HTTP payload extraction
- cookies/headers/status
- response shape for clients

Business auth logic is not here.

## 3) Action definition is in auth contributor

```js
// packages/auth/auth-provider-supabase-core/src/actions/auth.contributor.js
{
  id: "auth.login.password",
  version: 1,
  kind: "command",
  channels: ["api", "internal"],
  surfaces: ["app", "admin", "console"],
  visibility: "public",
  inputSchema: OBJECT_INPUT_SCHEMA,
  permission: allowPublic,
  idempotency: "none",
  async execute(input) {
    return authService.login(input);
  }
}
```

## 4) Session read flow (`GET /api/session`)

Controller calls `auth.session.read` action, then applies cookie updates if needed.

```js
// auth-fastify-adapter controller
const authResult = await executeAction(actionExecutor, {
  actionId: AUTH_ACTION_IDS.SESSION_READ,
  request
});

if (authResult.clearSession) {
  authService.clearSessionCookies(reply);
}
if (authResult.session) {
  authService.writeSessionCookies(reply, authResult.session);
}
```

And in contributor:

```js
{
  id: "auth.session.read",
  kind: "query",
  permission: allowPublic,
  async execute(_input, context) {
    return authService.authenticateRequest(requireRequestContext(context, "auth.session.read"));
  }
}
```

## 5) Where request actor is attached

Auth policy plugin resolves actor before protected handlers run.

```js
// server/fastify/auth.plugin.js
async resolveActor(request, reply) {
  const authResult = await authService.authenticateRequest(request);
  ...
}
```

This gives downstream request context:
- `request.user`
- `request.permissions`
- `request.workspace` when applicable

## 6) Common auth/session bugs

- Controller forgets cookie write/clear after action result
- Action uses wrong channel/surface restrictions
- Session endpoint changed response shape without matching schema/tests
