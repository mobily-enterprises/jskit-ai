# Q9: Security — env-driven framework extensions allow arbitrary module load

## Finding
- Location: `apps/jskit-value-app/server.js:92-95` passes `FRAMEWORK_EXTENSION_MODULES` to `loadFrameworkExtensions`, which blindly `import()`s resolved paths in `apps/jskit-value-app/server/framework/extensionsLoader.js:133-145`.

## Threat
If an attacker can influence env or `.env`, they can load arbitrary modules (code execution).

## Fix
Enforce an allowlist of extension directories, require a `.server.js` suffix, and reject paths outside a trusted root (`path.relative(root, resolved).startsWith("..")`). Optionally disable env-driven extensions in production unless explicitly allowed.
