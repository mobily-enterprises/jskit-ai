# Finding 002
- Severity: Medium
- Pattern type: Oversized service object; Single-responsibility violation
- File + line(s): `packages/billing/billing-service-core/src/shared/checkoutOrchestrator.service.js:1`, `packages/billing/billing-worker-core/src/shared/reconciliation.service.js:1`
- Why it can drift/fail: Both files explicitly disable `max-lines`, and each concentrates many responsibilities (normalization, orchestration, provider interaction, reconciliation, retry policy) into a single module. This makes the code harder to reason about, harder to test in isolation, and more likely to accumulate subtle regressions as it grows.
- Concrete impact: High change-risk areas become “hot spots” where small edits can unintentionally alter other code paths, and test coverage becomes expensive to maintain.
- Exact remediation: Split each service into smaller modules by responsibility (e.g., normalization/helpers, orchestration, provider adapters, reconciliation policy). Reinstate `max-lines` to enforce modularity. Add focused unit tests for each sub-module.
