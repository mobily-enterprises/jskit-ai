# Q4: Convenient lies — Paddle SDK JSON parse errors swallowed

## Finding
- Location: `packages/billing/billing-provider-paddle/src/shared/sdk.service.js:438`
- Behavior: JSON parse errors are swallowed and replaced with `{}`.

## Why risky
Provider errors lose the real cause (invalid JSON vs API error), complicating debugging.

## Better error strategy
Parse with fallback to raw text and include `parseError`/`rawBody` in mapped error metadata.
