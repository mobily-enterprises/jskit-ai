# Q4: Convenient lies — alert realtime publish failure hidden

## Finding
- Location: `apps/jskit-value-app/server/modules/alerts/service.js:213-236`
- Behavior: Realtime publish failure is converted to `false` with no log/metric; API still succeeds.

## Why risky
Clients believe alerts propagated when realtime publish failed; no trace for troubleshooting.

## Better error strategy
Emit a warning/metric with alert ID and payload, or return `realtimePublished:false` so callers can react or retry.
