# Fastify Auth Policy Package Plan (`@jskit/fastify-auth-policy`)

Status: Proposed  
Date: 2026-02-23  
Difficulty: 9

## 1. Objective

Provide reusable Fastify auth/policy wiring that standardizes route enforcement without coupling to a specific app identity model.

## 2. Non-Negotiable Boundaries

1. No direct DB queries in package.
2. No hardcoded claims schema.
3. No hardcoded workspace model.
4. No app-specific permissions in package code.

## 3. Scope

## In scope

1. Route metadata-driven policy checks.
2. Plugin hooks for auth/workspace/permission requirements.
3. Common error response mapping.
4. Fastify decorator helpers for policy metadata.

## Out of scope

1. Token verification implementation.
2. Membership repository logic.
3. App-specific route policy content.

## 4. Package Structure

```text
packages/auth/fastify-auth-policy/
  package.json
  README.md
  src/
    index.js
    plugin.js
    routeMeta.js
    errors.js
    contracts.js
  test/
    plugin.test.js
    routeMeta.test.js
```

## 5. Public API

```js
export function authPolicyPlugin(deps, options);
export function withAuthPolicy(meta);
export function mergeAuthPolicy(routeOptions, meta);
```

Injected dependencies:

1. `resolveActor(req)`
2. `resolveContext(req)`
3. `hasPermission(input)`
4. `onPolicyDenied(context)` (optional)

## 6. Migration Plan

## Phase 1

1. Extract metadata model and pure policy evaluators.
2. Keep existing plugin in app temporarily.

## Phase 2

1. Move Fastify plugin wiring into package.
2. Inject app-specific auth/context resolvers.

## Phase 3

1. Migrate all protected route registration.
2. Remove old plugin implementation.

## 7. Test Plan

1. Metadata precedence rules.
2. Require-auth/require-context/require-permission combinations.
3. Error mapping consistency.
4. Dependency-injection failure behavior.

## 8. Acceptance Criteria

1. Package does not know app identity schemas.
2. All auth behavior preserved through dependency adapters.
3. Route policy remains declarative and testable.

## 9. Versioning Rules

1. Patch: bugfixes.
2. Minor: additive policy metadata options.
3. Major: plugin contract changes.

## 10. Anti-Abstraction Guardrails

1. No custom policy DSL language.
2. No hidden magic inference from route paths.
3. Keep explicit route metadata.

## 11. Work Breakdown

1. Scaffold package.
2. Extract contracts.
3. Extract plugin.
4. Add tests.
5. Migrate app routes.

## 12. Definition of Done

1. Fastify policy plumbing centralized in package.
2. App-specific auth logic only in injected deps.
3. Route auth tests pass unchanged.
