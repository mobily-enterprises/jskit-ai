# HTTP Client Runtime Package Plan (`@jskit/http-client-runtime`)

Status: Proposed  
Date: 2026-02-23  
Difficulty: 6

## 1. Objective

Provide a shared frontend HTTP transport runtime with consistent retry, auth headers, correlation IDs, and error normalization.

## 2. Non-Negotiable Boundaries

1. No app endpoint definitions.
2. No UI framework coupling.
3. No app-specific header names hardcoded without configuration.
4. No app-specific error message mapping.

## 3. Scope

## In scope

1. Request execution wrapper over `fetch`.
2. Optional auth/csrf token providers.
3. Retry-once policy hooks (e.g., csrf refresh).
4. Correlation/command-id hooks.
5. Standardized error object shape.

## Out of scope

1. API endpoint method wrappers.
2. Query cache integration.
3. UI toast/notification behavior.

## 5. Public API

```js
export function createHttpClient(options);

client.request({ method, path, query, body, headers, signal });
client.get(path, options);
client.post(path, body, options);
```

Options are callback-driven:

1. `getAuthToken()`
2. `getCsrfToken()`
3. `onUnauthorized(error)`
4. `onRetryableFailure(context)`
5. `getCorrelationId()`

## 6. Migration Plan

## Phase 1

1. Extract existing generic transport logic.
2. Keep defaults aligned with current behavior.

## Phase 2

1. Migrate one API module first.
2. Migrate all API modules gradually.

## Phase 3

1. Remove legacy transport file.
2. Keep app-local endpoint wrappers intact.

## 7. Test Plan

1. Header injection correctness.
2. Query/body serialization.
3. Retry behavior boundaries.
4. Error normalization for network/http failures.
5. Abort signal support.

## 8. Acceptance Criteria

1. All frontend API calls go through shared runtime.
2. No endpoint/domain specifics in package.
3. Behavior parity for auth/csrf flows.

## 9. Versioning Rules

1. Patch: bugfix.
2. Minor: additive options.
3. Major: request/response/error API breaking changes.

## 10. Anti-Abstraction Guardrails

1. Do not add API resource clients to this package.
2. Keep package to transport concerns only.
3. Keep retries explicit and bounded.

## 11. Work Breakdown

1. Scaffold package.
2. Port core client.
3. Add tests.
4. Migrate imports.
5. Remove old transport implementation.

## 12. Definition of Done

1. Shared transport runtime in use by all web API modules.
2. Existing behavior and tests preserved.
3. Package remains domain-neutral.
